Executive Summary

The system accepts any request type, applies rules to find approvers, notifies automatically, and records every decision until it finishes (Approved, Rejected, or Canceled).

How It Works (Plain English)

  1. Select the type (Vacation, Purchase, etc.).
  2. Enter details and upload files if needed.
  3. The system picks approvers using rules (e.g., SUPERVISOR).
  4. First approver is notified via email or Teams.
  5. Approve / Reject; if approved and there are more, it continues.
  6. It ends: Approved (all OK), Rejected (someone said NO), or Canceled (you stopped it).
  7. The related module is updated (e.g., leave marked as approved).

Approver Rules (Who approves)

Instead of picking people one by one, we use texts that “mean” a specific person.

If a rule doesn’t find a valid person, that step is skipped. If no rule resolves any approver, the request isn’t created.

Statuses

Request

  • Pending: there are still approvers left.
  • Approved: everyone approved.
  • Rejected: someone said NO.
  • Canceled: creator stopped it.

Individual Step

  • Approved

What is a Webhook?

A “digital doorbell”. When something happens (new request, change, final decision), the system rings Power Automate or Teams. They can show cards or buttons so you can approve right there.

Files & Security

Files are stored internally. To share them we generate temporary links with expiry (like keys that later lock), preventing unwanted access.

FAQ

How do I know who is approving now?

The screen shows the current approver, and notifications do too.

Can I cancel?

Yes, while it’s Pending and you are the creator.

What if my manager changed?

New requests recalculate. A pending one can be Reassigned.

Can I approve from my phone?

Yes, via the Teams card or email link.

Is the rejector tracked?

Everything is recorded with date and comment.

Glossary

  • Request: what you submit (vacation, purchase).
  • Step: a single approval decision.
  • Rule: text that defines who will approve.
  • Webhook: automatic notification to another system.
  • Reassign: change the approver for a step.
  • Attachments: uploaded files.
  • Comments: approval/rejection notes.
